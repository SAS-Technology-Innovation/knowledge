## SAS WiFi Network

All students and staff can connect to the secure SAS wireless network using their SAS credentials. The SAS WiFi network uses **role-based access control**, automatically providing appropriate access to resources based on your account type.

<Note>
  **One Network, All Devices**: Use the same SAS username and password for WiFi that you use for Google and ClassLink.
</Note>

## WiFi Credentials

<Steps>
  <Step title="Network Name">
    Look for the network: **SAS**
  </Step>
  <Step title="Username">

## Connecting Your Device

<Tabs>
  <Tab title="iPad">
    **Connect iPad to SAS WiFi:**

    1. Open **Settings** app
    2. Tap **Wi-Fi**
    3. Make sure Wi-Fi is **ON** (toggle should be green)
    4. Select network: **SAS**
    5. Enter your **SAS username** (without @sas.edu.sg)
    6. Enter your **SAS password**
    7. Tap **Join**
    8. Accept any security certificates if prompted

    <Tip>
      **Auto-Connect**: Once connected, your iPad will automatically connect to SAS WiFi when in range!
    </Tip>
    **First Time Only:**

    - You may need to accept a certificate
    - Trust the "SAS" certificate when prompted
  </Tab>
  <Tab title="MacBook (macOS)">
    **Connect MacBook to SAS WiFi:**

    1. Click the **WiFi icon** in the menu bar (top right)
    2. Select network: **SAS**
    3. Enter your **SAS username** (without @sas.edu.sg)
    4. Enter your **SAS password**
    5. Click **Join**
    6. If prompted, accept the security certificate

    <AccordionGroup>
      <Accordion title="Troubleshooting macOS Connection" icon="apple">
        **If connection fails:**

        1. **Forget the network:**
           - System Settings ‚Üí Network ‚Üí WiFi ‚Üí Advanced
           - Select "SAS" ‚Üí Click minus (-) to remove
           - Try connecting again
        2. **Check credentials:**
           - Username must be WITHOUT @sas.edu.sg
           - Password is case-sensitive
        3. **Update macOS:**
           - System Settings ‚Üí General ‚Üí Software Update
           - Install any available updates
      </Accordion>
    </AccordionGroup>
  </Tab>
  <Tab title="Windows Laptop (BYOD)">
    **Connect Windows PC to SAS WiFi:**

    1. Click **WiFi icon** in system tray (bottom right)
    2. Click **SAS**
    3. Click **Connect**
    4. Enter your **SAS username** (without @sas.edu.sg)
    5. Enter your **SAS password**
    6. Click **OK**
    7. Accept certificate if prompted

    **Advanced Settings (if needed):**

    - Security type: WPA2-Enterprise
    - Encryption: AES
    - Authentication: PEAP or TTLS
  </Tab>
  <Tab title="Chromebook (BYOD)">
    **Connect Chromebook to SAS WiFi:**

    1. Click **time** in bottom right corner
    2. Click **WiFi** icon (or Settings)
    3. Select network: **SAS**
    4. **EAP method**: PEAP
    5. **Phase-2 authentication**: MSCHAPV2
    6. **Identity**: Your SAS username (without @sas.edu.sg)
    7. **Password**: Your SAS password
    8. Click **Connect**

    **Certificate:**

    - Select "Do not check" or accept SAS certificate
  </Tab>
  <Tab title="Personal Devices">
    **Connect Phone/Tablet to SAS WiFi:**

    **iPhone/Android Phone:**

    1. Open **Settings** ‚Üí **WiFi**
    2. Select **SAS**
    3. Enter **SAS username** (without @sas.edu.sg)
    4. Enter **SAS password**
    5. Tap **Join/Connect**
    6. Accept certificate if prompted

    <Warning>
      **Personal Device Policy**: Check with IT about acceptable use of personal devices on SAS network.
    </Warning>
  </Tab>
</Tabs>

## Troubleshooting

<AccordionGroup>
  <Accordion title="WiFi won't connect" icon="circle-exclamation">
    **Try these steps in order:**

    1. **Verify credentials:**
       - Username WITHOUT @sas.edu.sg ‚úì
       - Password is correct and case-sensitive ‚úì
       - No extra spaces in username or password ‚úì
    2. **Forget and reconnect:**
       - Remove/forget the SAS network
       - Turn WiFi off and on
       - Search for SAS network again
       - Re-enter credentials
    3. **Restart device:**
       - Completely power off your device
       - Wait 10 seconds
       - Power back on
       - Try connecting again
    4. **Check WiFi is working:**
       - Can others connect to SAS WiFi?
       - Try SAS-Guest to test connection
       - Check if WiFi is working in other areas

    **Still not working?** Contact [ithelp@sas.edu.sg](mailto:ithelp@sas.edu.sg)
  </Accordion>
  <Accordion title="Connected but no internet" icon="globe-slash">
    **Troubleshooting steps:**

    1. **Verify connection:**
       - Check WiFi icon shows connected
       - Look for "No Internet" message
       - Try opening a webpage
    2. **Renew connection:**
       - Disconnect from WiFi
       - Wait 5 seconds
       - Reconnect to SAS network
    3. **Clear DNS cache:**
       - **macOS**: Terminal ‚Üí `sudo dscacheutil -flushcache`
       - **Windows**: Command Prompt ‚Üí `ipconfig /flushdns`
       - **iPad/iPhone**: Restart device
    4. **Try different website:**
       - Some sites may be blocked
       - Try google.com or sas.edu.sg

    **Persistent issues?** Email [ithelp@sas.edu.sg](mailto:ithelp@sas.edu.sg)
  </Accordion>
  <Accordion title="Keeps disconnecting" icon="rotate">
    **Stability issues:**

    1. **Check signal strength:**
       - Move closer to access point
       - Avoid interference (microwaves, thick walls)
       - Try a different location
    2. **Update device:**
       - Install latest iOS/macOS/Windows updates
       - Update network drivers (Windows)
    3. **Forget other networks:**
       - Remove old/unused WiFi networks
       - Prevent auto-switching between networks
    4. **Device-specific:**
       - iPad: Settings ‚Üí General ‚Üí Reset ‚Üí Reset Network Settings
       - MacBook: Remove network preferences and reconnect

    **Ongoing problems?** Contact [ithelp@sas.edu.sg](mailto:ithelp@sas.edu.sg)
  </Accordion>
  <Accordion title="Certificate errors or warnings" icon="certificate">
    **Security certificate issues:**

    **What it means:**

    - SAS WiFi uses security certificates
    - Your device needs to trust the SAS certificate
    - This is normal and safe

    **What to do:**

    1. When prompted, click **Trust** or **Accept**
    2. Certificate name should show "SAS" or similar
    3. If unsure, contact IT before accepting

    **Don't see certificate prompt?**

    - May have been accepted previously
    - Connection should work normally

    **Certificate rejected?** Contact [ithelp@sas.edu.sg](mailto:ithelp@sas.edu.sg)
  </Accordion>
  <Accordion title="I forgot my password" icon="key">
    **Password reset:**

    **For Students:**

    1. Ask your teacher for help
    2. Visit the IT Help Desk with student ID
    3. Parents can email [ithelp@sas.edu.sg](mailto:ithelp@sas.edu.sg)

    **For Staff:**

    1. Use the self-service password reset portal
    2. Contact [ithelp@sas.edu.sg](mailto:ithelp@sas.edu.sg)
    3. Visit IT Help Desk with staff ID

    <Warning>
      Changing your SAS password updates it for WiFi, Google, AND ClassLink.
    </Warning>
  </Accordion>
</AccordionGroup>
